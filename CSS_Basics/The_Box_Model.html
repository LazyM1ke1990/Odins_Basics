CSS: The Box Model

The box model is the first most important step to being successful in CSS. Everything on the web page is a box. You can have boxes inside a box or side by side each other.
These boxes can be roughly understood by using the 'Outline' Property on all elements (Using the Universal selector). The boxes wont be visible unless you give a border, border thickness and border color value.

Result:The red borders around he boxes to make them visable

The size, shape and space around them can be manipulated in many different ways using 'Padding','Margin' and 'Borders'

  • Padding: Is the empty space between the border of the box and the contents of the box. This space can be manipulated to give more or less empty space between the box contents and box border
  • Margin: Is the space outside of the box border and between the adjacent boxes border. This increases or decreases distance between the 2 boxes.
  • Border: Is the space between the margin and the padding. Its what creates the border effect around a box. It can be as little as a pixel thick(Very thin)

Standard/Alternative box Model

The box model consists of a standard Variant model and an Alternative variant. These variants use the same Content,Margin,Padding,Border box measurements but style them differently when visible in the browser.
Using the below dimesions as an example we can workout the actual sizes of each box which is done in the below sub headings.



  • Standard - If you set a height and width property of an element in a standard box variant (Which is the browsers default) these values will define the contents box dimensions only, disregarding the padding,border and margin.

    The actual space taken up by this standard variant box is 410px wide and 210px high this is because the content box width is 350px plus 25px of padding either side of the content, then 5px border either side of the padding
    (350 + 25 + 25 + 5 + 5= 410). Same thing for the height (150 + 25 + 25 + 5 + 5= 210).



  • Alternative - If you set the height and width property in an alternative box variant (Which needs to be defined using within CSS styling) Then the total box dimesions will stay as
    the height and width values that were defined including the padding and border values. This is a benefit as you do not need to calculate the dimesions manually. They are what you set them as.
